# Auto Generate Dalton Articles

Generate Dalton Bioanalytics scientific summary articles with Claude Code. The app can run once immediately or run immediately and then continue on a recurring day interval.

## Quick Setup

1. Install Claude Code CLI.

2. Confirm Claude works from PowerShell:

```powershell
claude --version
```

3. Download the latest Windows release:

https://github.com/Xinye-Star-Yu/Auto_generate_dalton_articles/releases

4. Download and unzip:

```text
AutoGenerateDaltonArticles-windows.zip
```

5. Put the unzipped app somewhere permanent, for example:

```text
C:\Users\xinye\Apps\AutoGenerateDaltonArticles\
```

6. Double-click:

```text
AutoGenerateDaltonArticles.exe
```

## Using the App

The GUI has one setting:

```text
Run every [X] days
```

Then click `Start`.

- `0` days: run one article immediately and do not create a schedule.
- `1` day: run one article immediately, then run again every day.
- `7` days: run one article immediately, then run again every 7 days.

Click `Stop` to remove the recurring schedule.

Logs appear in the GUI while it runs.

## Where Outputs Go

Outputs are created next to `AutoGenerateDaltonArticles.exe`.

Example app folder:

```text
C:\Users\xinye\Apps\AutoGenerateDaltonArticles\
```

Generated article folders appear there:

```text
C:\Users\xinye\Apps\AutoGenerateDaltonArticles\2026-04-29-1\
C:\Users\xinye\Apps\AutoGenerateDaltonArticles\2026-04-29-2\
```

Each article folder contains:

```text
article_output.pdf
figure1.jpg
```

or another Figure 1 image extension, such as `.png`.

The app also keeps these files next to the exe:

```text
scheduler.log
scheduler_config.json
Duplication_Check_Dalton.csv
template_output.pdf
```

`Duplication_Check_Dalton.csv` stores used DOIs so future runs avoid duplicate articles.

## Requirements

For the Windows release:

- Windows
- Claude Code CLI installed and available as `claude`
- A Claude Code account/session that can run prompts
- Internet access for article search and figure download

Optional:

- Poppler `pdftotext` on PATH for stricter PDF validation

## Important Notes

- The executable does not include Claude Code. Claude must be installed separately.
- Do not move the `.exe` after starting a recurring schedule. If you move it, click `Stop`, move it, then click `Start` again.
- Scheduled runs use Windows Task Scheduler with the task name `AutoGenerateArticleScheduler`.
- Article generation can take a while because Claude searches for a paper, downloads Figure 1, creates the PDF, updates the DOI CSV, and validates the output.

## Run From Source

Use this only if you want to run or edit the Python files directly.

```powershell
git clone https://github.com/Xinye-Star-Yu/Auto_generate_dalton_articles.git
cd Auto_generate_dalton_articles
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python scheduler_gui.py
```

Generate one article without the GUI:

```powershell
python generate.py
```

Scheduler CLI:

```powershell
python article_scheduler.py --interval-days 7 --run-time 09:00 --install
python article_scheduler.py --status
python article_scheduler.py --run-now
python article_scheduler.py --run-due
python article_scheduler.py --remove
```

## Build the EXE

Install build dependencies:

```powershell
pip install -r requirements.txt -r requirements-build.txt
```

Build:

```powershell
.\build_exe.ps1
```

Output:

```text
dist\AutoGenerateDaltonArticles.exe
```

## Create a GitHub Release

Tag a version:

```powershell
git tag v0.1.1
git push origin v0.1.1
```

GitHub Actions builds the Windows executable and attaches `AutoGenerateDaltonArticles-windows.zip` to the release.

## Project Files

- `scheduler_gui.py`: simple GUI
- `article_scheduler.py`: scheduler and recurring run logic
- `generate.py`: article generation workflow
- `Duplication_Check_Dalton.csv`: DOI history
- `template_output.pdf`: PDF template
- `build_exe.ps1`: local Windows exe build script
- `.github/workflows/release.yml`: GitHub release build workflow
